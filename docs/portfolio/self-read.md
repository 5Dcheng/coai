## 11. 简历摘要

可以写成：

> 负责 CoAI 项目认知层产品设计，针对 AI Coding 多轮开发中项目黑盒化、上下文丢失、功能边界漂移等问题，设计 feature cognition、mapper、anchor、current.md 等机制，将功能目标、输入输出、关键流程、约束边界与真实源码入口连接，形成面向开发者和 agent 的项目认知地图。基于 TeamDesk Lite 真实 AI Coding 项目设计 baseline / AGENTS.md / CoAI 三组对照实验，CoAI 在新窗口项目认知任务中将源码读取数从 26 降至 8（-69.2%），GLM billed usage 从 299,655 降至 174,740 tokens（-41.7%）；在 R26-R30 维护任务中将源码相关读取从 46 降至 21（-54.3%）。通过实验验证 CoAI 可将 agent 从全局源码探索引导至基于功能认知的定向源码验证，提升项目认知恢复、功能定位和长期维护连续性。

## 12. 面试讲述建议

推荐讲法：

1. **背景**：AI coding 让代码生成变快，但项目认知衰减更快。
2. **问题抽象**：本质不是缺文档，而是缺少可随代码演进维护的项目认知层。
3. **产品方案**：CoAI 用 feature cognition + mapper/anchor + cognition backflow 建立功能语义到源码入口的连接。
4. **对照验证**：baseline、AGENTS.md、CoAI 三组实验。
5. **结果**：CoAI 提供项目认知地图，并在测试中减少重复源码探索。
6. **边界**：CoAI 不替代源码验证，也不承诺所有任务都更省；它的价值在长期维护和跨 chat 认知延续。

## 13. 面试追问准备

### Q1：为什么 AGENTS.md 不够？

AGENTS.md 是一个强 baseline，但它是手动维护的单文件上下文资产。它生成和更新都需要成本，也容易随项目推进而过期。CoAI 的差异在于 feature-level 组织、mapper/anchor 源码入口和认知回流机制，更适合长期维护。

### Q2：源码读取下降是不是说明不读源码？

不是。源码仍然是事实来源。下降说明 agent 不再从全局源码盲扫开始，而是先读取项目认知地图，再定向验证关键源码。CoAI 的价值是让源码阅读更有目标。

### Q3：CoAI 最大的产品风险是什么？

认知资产也可能过期。因此 CoAI 必须嵌入开发循环：改功能前读 feature cognition，改代码时维护 mapper/anchor，改完后回流稳定认知。

### Q4：这个项目最体现产品经理能力的地方是什么？

不是单纯做工具，而是从 AI Coding 的真实协作问题中抽象出产品机制，设计替代方案对照实验，并用数据修正产品定位。
